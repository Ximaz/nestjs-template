# Prisma & PostgreSQL

The `prisma` module owns the database connection. `PrismaService` extends `PrismaClient` directly so every query helper is available on the injected service.

---

## Database service

Declared in [`docker-compose.yml`](../docker-compose.yml):

| Aspect          | Value                                                    |
| --------------- | -------------------------------------------------------- |
| Image           | `postgres:16.8-alpine3.20`                               |
| Container       | `project-database`                                       |
| Hostname        | `database`                                               |
| Network         | `project-network`                                        |
| Volume          | `project-database` mounted at `/var/lib/postgresql/data` |
| Ports           | `5432:5432` (published — useful for psql / Prisma Studio)|
| Healthcheck     | `pg_isready -U $POSTGRES_USER -d $POSTGRES_DB`           |
| Shared memory   | `shm_size: 128mb`                                        |

### Secrets — `secrets/database/.env`

| Variable            | Description           |
| ------------------- | --------------------- |
| `POSTGRES_USER`     | DB superuser name     |
| `POSTGRES_PASSWORD` | DB superuser password |
| `POSTGRES_DB`       | Default database name |

## Backend integration

[`packages/backend/src/prisma/prisma.service.ts`](../packages/backend/src/prisma/prisma.service.ts) extends the generated `PrismaClient` using the `@prisma/adapter-pg` adapter:

```ts
@Injectable()
export class PrismaService extends PrismaClient {
  constructor(configService: ConfigService) {
    const adapter = new PrismaPg({
      connectionString: configService.getOrThrow<string>('DATABASE_URL'),
    });
    super({ adapter });
  }
}
```

`PrismaModule` exports the service for injection by other modules (`AuthModule`, `OauthModule`, `UsersModule`).

### `DATABASE_URL`

In `secrets/backend/.env`:

```
DATABASE_URL="postgresql://<user>:<password>@<host>:5432/<db>"
```

- From the backend container on `project-network`: host = `database`.
- From the host (`pnpm start:dev`): host = `localhost`.

## Schema

[`packages/backend/prisma/schema.prisma`](../packages/backend/prisma/schema.prisma) defines three models with one-to-one relations and cascade deletes:

```
User ─┬─ LocalCredential (optional)
      └─ OAuthCredential (optional)
```

| Model             | Notes                                                                |
| ----------------- | -------------------------------------------------------------------- |
| `User`            | `id` is `cuid2`. Unique `email`. Optional `firstName`, `lastName`, `picture`. |
| `LocalCredential` | `email` (unique), `hashedPassword` (argon2). Cascade delete from `User`. |
| `OAuthCredential` | `(provider, sub)` unique index. Cascade delete from `User`.          |

All tables and most fields use `@@map` / `@map` to snake_case in the database while keeping camelCase in TypeScript.

The generated client output lives in [`packages/backend/src/generated/prisma/`](../packages/backend/src/generated/prisma/) (gitignored — regenerated by `prisma:generate`).

## Workflow

All scripts run via `pnpm --filter @project/backend <script>`:

| Script            | What it does                                       |
| ----------------- | -------------------------------------------------- |
| `prisma:generate` | Regenerate the Prisma client into `src/generated/` |
| `prisma:migrate`  | Create & apply a migration in dev (`migrate dev`)  |
| `prisma:deploy`   | Apply pending migrations (prod-style)              |
| `prisma:reset`    | Drop everything and re-apply (destructive)         |
| `prisma:studio`   | Open Prisma Studio against the current DB          |
| `prisma:format`   | Format `schema.prisma`                             |

First-time bootstrap:

```bash
docker compose up -d database
pnpm --filter @project/backend prisma:generate
pnpm --filter @project/backend prisma:migrate
```

The Docker build runs `prisma:generate` in the `build` stage, so the runtime image ships with a generated client.
